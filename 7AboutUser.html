     $ who : 이 컴퓨터에 누가 접속해있는가

     ~$로 끝나면 일반 사용자
     ~#로 끝나면 root 사용자

     sudo (super user do)
     리눅스는 다중 사용자 운영체제
     사용자마다 어떤 일에 대한 권한이 있음.
     중요한 일에 대한 권한은 super user(root)만 가지고 있음.
     따라서 super user의 권한이 필요할 때 sudo를 사용한다.
     직접 root계정에 접속해서 명령어를 실행 시킬 수도 있지만, 안전을 위해 일반 사용자로 로그인 후 sudo를 사용하도록 하자.

     사용자 추가
     $ sudo useradd -m [사용자명]
     -> /home에 가보면 사용자가 추가된 걸 볼 수 있음.
     -> 그러나, 이렇게 만들면 해당 계정에서 sudo 명령어를 사용할 수 없음.

     sudo 사용할 수 있도록 변경
     $ sudo usermod -a -G sudo [사용자명]

     비밀번호 설정
     $ sudo passwd [사용자명]

     로그인 (su 사용)
     $ su - [사용자명]

     모든 사용자 로그인 막기
     # touch /etc/nologin
     -> /etc에 nologin 파일을 생성하면 현재 접속 관리자를 제외한 나머지 모든 계정은 접근 불가
     -> 만약 이 파일에 내용을 작성하면 FTP 또는 TELNET 접속 사용자는 해당 파일 내용 확인 가능

     특정 사용자 로그인 막기
     # vi/etc/shadow
     -> testuser:*: PW * 처리 (* 표시를 한 사용자는 로그인 불가)

     가입자 mysql 실행 권한 주기
     # vi /etc/skel/.bash_profile

     PATH1=$PATH:/$HOME/bin  <<설정 부분
     PATH=$PATH1:/usr/local/mysql/bin <<설정 부분
     BASH_ENV=$HOME/.bashrc
     USERNAME=""
     export USERNAME BASH_ENV PATH1 PATH <<설정 부분

     사용자 로그인시 공지사항 알리기
     # vi /etc/issue.net : 로그인 전 알릴내용 적기
     # vi /etc/motd : 공지사항 내용 입력하기
