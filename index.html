      세미콜론(;)
      하나의 명령어 라인에서 여러개의 명령을 실행가능
      첫 번쨰 명령이 실패해도 두번째 명령이 실행됨.

      파이프(|)
      프로세스의 통신을 위해 도입한 것으로 어떤 프로세스의 표준 출력이 다른 프로세스의
      표준 입력으로 쓰이게 함.(앞의 실행한 명령어의 출력 결과를 뒤에 실행하는 명령의 입력값으로)

      *셀 프롬프트에서는 개수 제한이 없고 주로 awk,cat,grep,more 등과 함께 사용.
      ex)$ cat.bashrc | more

      더블 엔드 퍼센트(&&)
      다중 명령을 한 프 롬프트 안에서 실핼할 수 있게 함.
      첫 번째 명령이 오류 없이 실행되었을 때만 두 번째 명령을 실행한다는 점에서 세미콜론과 차이가 있음.

      더블 버티컬 바(||)
      첫 번째 명령에서 오류 가 발생할 경우 뒤의 명령을 수행.

      ctrl+ l
      clear와 동일하나, vi 작업할때 백그라운드에서 실행 중인 프로세스가 뱉는 메시지를 지울때…

      ctrl + r
      과거 입력했는 명령중에 유사한 명령 추천

      cd *kong
      마지막에 kong으로 끝나는 디렉토리로 진입

      cd –
      바로 이전에 있던 디렉토리로 복귀, cd -2 같은걸로 스크립트 응용가능

      ls k[on]g
      k로 시작하고 중간에 o나 n를 포함하고 g로 끝나는 파일을 출력

      cp kong.txt{,-cute}
      앞에서 입력한 파일명에다가 -cute을 붙여서 파일 생성

      \vi
      alias로 실행하지 않고 그냥 명령을 실행, alias vi=’vim’으로되어 있다면 그냥 vi가 실행됨
